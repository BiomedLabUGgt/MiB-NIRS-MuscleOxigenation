1/**
2 * @file I2C.h
3 * @brief I2C1 Master Communication Driver (400 kHz, STM32F303K8)
4 * @details Low-level I2C driver for sensor communication using open-drain configuration.
5 *
6 * ### Hardware Configuration
7 * - **Peripheral**: I2C1 (APB1 @ 32 MHz)
8 * - **Pins**: PB6 (SCL), PB7 (SDA) - open-drain outputs
9 * - **Speed**: 400 kHz (Fast-mode compliant)
10 * - **Addressing**: 7-bit slave addressing (MSB first)
11 * - **Protocol**: Master-only; repeated START supported for register read
12 *
13 * ### Timing (400 kHz mode, APB1 = 32 MHz)
14 * - SCL period: 2.5 µs
15 * - SCL high time: ~1.5 µs
16 * - SCL low time: ~1.0 µs
17 * - Setup/hold times: Per I2C Fast-mode specification
18 *
19 * ### Driver Characteristics
20 * - **Write latency**: ~30-50 µs per byte (2 bytes minimum per transaction)
21 * - **Read latency**: ~100-150 µs per byte (repeated START adds ~50 µs overhead)
22 * - **Blocking**: Yes (waits for bus/flags; no interrupts or DMA)
23 * - **Thread-safe**: No (not safe for concurrent I2C accesses)
24 *
25 * ### Supported Transactions
26 * 1. **Write**: Master writes register address + 1 data byte (MAX30101 registers)
27 * 2. **Read**: Master writes address, repeated START, reads N bytes (FIFO streaming)
28 *
29 * @author Julio Fajardo
30 * @date 2024-06-01
31 * @version 2.0
32 * @note For STM32F303K8 only. TIMINGR value 0x00C50F26 is specific to APB1 = 32 MHz
33 * @todo Add error handling (NAK detection, bus timeout)
34 * @todo Implement DMA for high-speed FIFO reads
35 */
